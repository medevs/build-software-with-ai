# Your First Week — Checklist

Lesson 30 as a simple list to check off, day by day. Pick one real project — new or existing, it doesn't matter — and work through the seven days on it. Don't rush it into one sitting: let each day's habit settle before adding the next, and if a day slips, repeat it rather than skip ahead. The habit matters more than the calendar.

## Day 1 — Pick a tool, write lean rules

- [ ] Choose one coding agent and commit to it for the whole week.
- [ ] Write a short rules file: a few sentences about what the project is and
      how it's structured.
- [ ] Add two or three things the agent should always or never do.
- [ ] Keep the whole file under a page.

## Day 2 — Run one full loop

- [ ] Pick the smallest real task you can find.
- [ ] Research: read the relevant code.
- [ ] Plan: write a short plan.
- [ ] Implement: let the agent build from the plan.
- [ ] Validate: check the result yourself before trusting it.

## Day 3 — Add your first skill

- [ ] Notice a step you repeated on Day 2 — priming context, writing a plan,
      running validation.
- [ ] Turn it into a reusable skill or saved workflow so you don't retype it
      next time.

## Day 4 — Add one guardrail

- [ ] Pick one mistake you don't want repeated — an accidental destructive
      command, a skipped test run.
- [ ] Add a simple check that catches it automatically, instead of relying on
      your memory.

## Day 5 — Run a second loop, faster

- [ ] Take on a slightly bigger task and run the full loop again.
- [ ] Notice how much less you have to explain this time — your rules file and
      skill already carry the context.

## Day 6 — Try parallel or remote

- [ ] Find two independent small tasks.
- [ ] Run them as two separate loops at once instead of one after another.
- [ ] Feel the difference between doing the typing and directing the work.

## Day 7 — Review and refine

- [ ] Look back at everything you wrote this week.
- [ ] Cut anything in your rules file that turned out to be unnecessary.
- [ ] Write down the one rule you wish you'd had on Day 1.

---

One throughline to keep: **the bottleneck was never the model — it's the
system around it.** The tool you picked on Day 1 is replaceable; the habits
you built around it are not. You don't need a perfect system on day one — you
need a working loop you run again tomorrow.
